# Development Rules

1.  Never generate placeholder code.
2.  Every feature must include frontend, backend, API, database, tests
    and documentation.
3.  Build incrementally.
4.  No duplicated logic.
5.  Every commit must be deployable.
6.  Store secrets only in Azure Key Vault.
7.  Every API must include validation, authentication and authorization.
8.  Everything must run in Docker.
9.  Everything must support Kubernetes.
10. Update README after every milestone.
